## 📋 Prerequisites — OOP Fundamentals

Before diving into OOD, make sure you have a solid understanding of the four pillars of OOP:

> **Encapsulation · Abstraction · Inheritance · Polymorphism**

**Recommended Language:** C++ or Java *(or any language you're comfortable with)*

The five essential design principles for writing maintainable and scalable object-oriented code.

### 🛠️ Practice Resources

| Resource | Type | Description |
|---|---|---|
| [Pluralsight Code Lab](https://www.pluralsight.com/codeschool) | Interactive Lab | Guided 54-min refactoring lab — fix a notification app using each SOLID principle |
| [Messy Tetris (GitHub)](https://github.com/ypisek/MessyTetris) | Messy Codebase | Purposely messy Tetris game — refactor it to follow SOLID principles |
| [PV260 SOLID Exercises (GitHub)](https://github.com/PV260/SOLID) | Exercises | University seminar problems + solutions in Java |
| [SOLID Principles in Go (GitHub)](https://github.com/tmrts/go-patterns) | Code Examples | Go-specific SOLID examples (bad vs. good) |
| [SOLID Principles in Python (GitHub)](https://github.com/Heather/clean-code-python) | Code Examples | Python-focused SOLID demonstrations |
| [Refactoring Guru — SOLID](https://refactoring.guru/solid) | Reference | Clear explanations with visual examples and code snippets |
| [ProProfs SOLID Quizzes](https://www.proprofs.com/quiz-school/story.php?title=solid-principles-quiz) | Quiz | Multiple-choice questions to test theoretical understanding |

### 📝 Practice Approach

1. **Learn** — Watch the playlist to understand each principle
2. **See** — Study the Refactoring Guru examples
3. **Break** — Run the messy codebase, understand what it does
4. **Fix** — Refactor one principle at a time (S → O → L → I → D)
5. **Verify** — Check your work against provided solutions
